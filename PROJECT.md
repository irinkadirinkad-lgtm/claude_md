### Доступы к YClients
Хранятся в Apps Script → Project Settings → Script Properties (НЕ в коде, НЕ в репозитории):
PARTNER_TOKEN, COMPANY_ID=32825, YC_LOGIN, YC_PASSWORD, USER_TOKEN.
Технический пользователь YClients создан специально для интеграции (не личный аккаунт).

### API YClients
- POST /api/v1/finance_transactions/{company_id} — создание операции
  поля: expense_id, amount, account_id, client_id, supplier_id, master_id, comment, date ("YYYY-MM-DD HH:MM:SS")
- Авторизация: Authorization: Bearer {partner_token}, User {user_token}
- user_token: POST /api/v1/auth {login, password}
- Вебхуки: событие «Финансовая операция получена» есть. Подключаются через регистрацию приватного приложения в кабинете разработчика (developers.yclients.com), не через старый раздел настроек. Публиковать в маркетплейс не нужно (тип — «Непубличное»).

### Автоматизация в Apps Script (уже реализовано, живёт только в Код.js — проверять там)
- Ежедневная отправка новых строк журнала в YClients (21:00), с учётом «Назначение» (пропускает строки «Только Планировщик»).
- Еженедельный перенос новых строк журнала в Планировщик (воскресенье 22:00), защита от задвоения — колонка «Отправлено в Планировщик».
- Автозакрытие месяца (1–3 число, 01:00) с самопроверкой архивации и email-уведомлением при сбое.
