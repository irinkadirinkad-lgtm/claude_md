## Yclients Finance App — две новые вкладки (В РАБОТЕ, начато 14.07)

Согласовано с Ириной 14.07. Делаем ОБЕ, начиная с «Аналитики дня».

**Вкладка 1 «Аналитика дня»** (данные — ЖИВЬЁМ из YClients по запросу, не фоном; движок
`collectRange_`/`previewDay_`/`previewRange_` в `finance-integration` уже есть):
- Выбор: один день (по умолч. сегодня) ИЛИ произвольный период (две любые даты).
- Мини-P&L дня: Доход − эквайринг(с услуг) − налог(7%) − себестоимость услуг − ФОТ мастеров = Чистая прибыль дня.
  ⚠️ Себестоимость услуг пока приблизительная («на клиента», детальная — Блок B). Налог/ФОТ = РАСЧЁТ (как в «Прибыли»).
- Клиенты дня: всего / новых, продажи товаров, средний чек.
- Мастера за день: по каждому — выручка, клиенты, услуги, ЗАРАБОТАЛ за день (по правилам: выход/%), загруженность мастера %.
- Загруженность салона % (по периоду — по дням). Источник: YClients `analytics/overall` `fullness_stats` + `charts/fullness_daily`.
  ⚠️ Формула заполненности у YClients не документирована — сверить на реальных данных, прежде чем доверять.
- Нагрузка по направлениям → детализация по услугам (`collectCategories_`).
- Касса: нал/безнал, эквайринг.
- Идея (согласовать): сравнение дня со средним по месяцу.

**Вкладка 2 «Клиенты»** (после «Сотрудников»; смотрим ПО МЕСЯЦАМ, актуальный подтягивается сам):
- Источник — СКРЫТЫЙ технический лист в таблице (интеграция пишет помесячно, приложение читает быстро;
  YClients лишний раз не дёргаем). Прошлые месяцы заморожены, текущий освежается на автопересчёте.
- Сводка месяца: всего, новых, вернувшихся, ПОСТОЯННЫХ, потерянных (кол-во + %), % удержания.
  ⚠️ Определения (решение Ирины): ПОСТОЯННЫЙ = приходит не реже раза в ~3 мес (разрыв до 3 мес — норма):
  ≥2 визита за последние 6 мес И последний визит ≤3 мес назад. ПОТЕРЯННЫЙ = был клиентом, но не приходил
  >3 мес. Считаем САМИ из `records` по датам визитов (не полагаемся на YClients `lost_count`, у него своя логика).
  Прочее (всего/новые/вернувшиеся/удержание) можно брать из `analytics/overall` `client_stats` и сверить.
- Траты (ОБЕЗЛИЧЕННО, без имён): самые большие общие траты / визиты / средние чеки.
- Помесячная динамика показателей.
- НЕ показываем: имена/фамилии, пол, возраст, ДР, категории/теги.

Архитектура: «Аналитика дня» — живой action у скрипта-интеграции (у него есть движок+доступы),
приложение зовёт по запросу. «Клиенты» — через скрытый лист + `finance-report-app-api` (как все вкладки).
Лимиты YClients: 200 запросов/мин, 5/сек — грузим один раз на выбранный день/период, кэш на сессию.

Пошагово: (1) бэкенд дня → сверка с программой → экран → сверка; (2) бэкенд клиентов (лист) → экран → сверка;
(3) уборка доков + merge в main.
